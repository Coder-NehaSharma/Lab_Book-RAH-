1. Introduction to Robotics

Robotics is where Engineering, Electronics, and Coding come together to create machines that can move, sense, and think! In this course, you will learn how to build your own robots from scratch.

What is a Robot?

A robot is a machine that can perform tasks automatically. It usually consists of three main parts:

1. Brain (Controller)

Thinking part. Examples: Arduino, Raspberry Pi, Micro:bit.

2. Body (Mechanical)

Moving parts. Examples: Motors, Wheels, Chassis, Arms.

3. Senses (Sensors)

Feeling part. Examples: IR Sensor (Eyes), Mic (Ears).

Applications of Robotics

  • üè• Medical: Surgical robots, prosthetics.
  • üöÄ Space: Mars Rovers like Curiosity and Perseverance.

5. 4x4 Keypad Interfacing

Keypads allow us to give secure inputs, like passwords or controlling robot direction manually.

How it works (Matrix Scanning)

The keypad has 4 Rows and 4 Columns. When you press a button, it connects a specific row to a specific column, and the Arduino detects which pair is connected.

Code Snippet: Password System

#include <Keypad.h>

const byte ROWS = 4;
const byte COLS = 4;
char keys[ROWS][COLS] = {
  {'1','2','3','A'},
  {'4','5','6','B'},
  {'7','8','9','C'},
  {'*','0','#','D'}
};
byte rowPins[ROWS] = {9, 8, 7, 6};
byte colPins[COLS] = {5, 4, 3, 2};

Keypad keypad = Keypad(makeKeymap(keys), rowPins, colPins, ROWS, COLS);

void loop() {
  char key = keypad.getKey();
  if (key){
    Serial.println(key); // Check Serial Monitor
  }
}

6. Sensors: The Robot's Senses

Just like humans have eyes and ears, robots use sensors to perceive the world.

IR Sensor (The Eye)

Use: Line following, proximity detection.

Working: Emits Infrared light; if it bounces back, an object is near.

Wiring: VCC(5V), GND, OUT(Pin 2)

Ultrasonic Sensor (HC-SR04)

Use: Measuring distance, avoiding obstacles.

Working: Sends sound waves like a bat.

Wiring: Trig(Pin 9), Echo(Pin 10)

PIR Sensor

Use: Motion detection (Burglar alarms).

Working: Detects heat (infrared) given off by living bodies.

Soil Moisture Sensor

Use: Smart Gardening.

Working: Measures resistance between two probes in soil.

Practical Activity: Obstacle Warning

Connect an Ultrasonic Sensor. If distance < 10cm, Turn on LED/Buzzer.

// Pseudo-code logic
if (distance < 10) {
    digitalWrite(ledPin, HIGH);
    tone(buzzer, 1000);
} else {
    digitalWrite(ledPin, LOW);
    noTone(buzzer);
}

L298N Motor Driver

Microcontrollers (Arduino) cannot power motors directly‚Äîthey are too weak! We use the L298N driver as a middleman.

Wiring Logic (H-Bridge)

The L298N has an "H-Bridge" inside, which allows it to swap the polarity of the motors, letting them spin Forward AND Backward.

Movement Code

// Motor A connections
int enA = 9;
int in1 = 8;
int in2 = 7;

void setup() {
  pinMode(enA, OUTPUT);
  pinMode(in1, OUTPUT);
  pinMode(in2, OUTPUT);
}

void loop() {
  // Move Forward
  digitalWrite(in1, HIGH);
  digitalWrite(in2, LOW);
  analogWrite(enA, 200); // Speed 0-255
  delay(2000);
  
  // Stop
  digitalWrite(in1, LOW);
  digitalWrite(in2, LOW);
  delay(1000);
}

8. Power System

A robot without power is just a statue. We need a safe and reliable energy source.

18650 Li-Ion Batteries

These are powerful rechargeable batteries used in modern robotics (and electric cars!).

  • Voltage: 3.7V per cell.
  • Pack: 2 cells in series = 7.4V (Perfect for Arduino & Motors).

‚ö†Ô∏è SAFETY WARNING

Never short-circuit batteries. They can get hot and catch fire! Always use a battery holder and turn off the switch when not in use.

12. Troubleshooting Guide

Robot not working? Don't panic! Check these common issues.

Symptom Possible Cause Solution
Code won't upload Wrong COM Port Check Tools > Port in Arduino IDE.
LCD Screen is blank Contrast Potentiometer Turn the blue knob on the I2C module back/forth.
Motors only hum but don't move Low Battery Charge batteries. USB power is not enough for motors.
Sensor gives random values Loose wiring Double-check jumper wires on the breadboard.
